# 모델 성능 평가

## 1. 학습 개요

제조/물류 AMR 안전 주행 보조 시스템을 가정하고, YOLOv8n 모델을 최종 데이터셋으로 파인튜닝했다.

학습 대상 클래스:

```text
0: person
1: box
2: traffic_cone
3: forklift
```

학습 산출물:

```text
runs/logistics_yolov8n/
  weights/
    best.pt
    last.pt
    best.onnx
  results.csv
  results.png
  confusion_matrix.png
```

## 2. 최종 성능 지표

학습 마지막 결과 기준 주요 지표는 다음과 같다.

| Metric | Value |
|---|---:|
| Precision(B) | 0.87 |
| Recall(B) | 0.72 |
| mAP50(B) | 0.79 |
| mAP50-95(B) | 0.56 |

## 3. 지표 해석

### Precision(B): 0.87

모델이 객체라고 예측한 결과 중 실제 정답인 비율이 높다. 즉, 오탐은 비교적 적은 편이다.

### Recall(B): 0.72

실제 객체 중 약 72%를 탐지했다. 일부 객체는 놓치는 경우가 있으며, 특히 데이터 수가 적거나 작은 객체에서 미탐 가능성이 있다.

### mAP50(B): 0.79

IoU 0.5 기준에서 객체 탐지 성능은 양호하다. 미니프로젝트 프로토타입 기준으로는 충분히 사용할 수 있는 수준이다.

### mAP50-95(B): 0.56

IoU 0.5부터 0.95까지 더 엄격한 기준으로 평균을 낸 지표다. bbox 위치가 매우 정확해야 높게 나오므로 mAP50보다 낮게 나오는 것이 일반적이다.

이번 프로젝트에서는 여러 공개 데이터셋을 병합했기 때문에 다음 요인이 mAP50-95를 낮출 수 있다.

- 데이터셋마다 bbox 라벨링 기준이 다름
- 촬영 환경과 객체 크기가 서로 다름
- `traffic_cone` 데이터가 많고 `forklift` 데이터가 적어 클래스 불균형이 있음
- YOLOv8n은 경량 모델이라 bbox 정밀도에서 한계가 있을 수 있음

## 4. 현재 결과 판단

현재 성능은 온디바이스 AIoT 미니프로젝트의 1차 프로토타입으로는 충분하다.

특히 `mAP50 = 0.79`이므로 객체 존재 여부와 클래스 탐지 자체는 양호하게 동작한다. `mAP50-95 = 0.56`은 모델이 실패했다는 의미가 아니라, bbox 정밀도와 데이터 품질을 개선할 여지가 있다는 의미로 해석한다.

## 5. 개선 방향

- `forklift` 데이터 추가 확보
- 클래스별 데이터 수 균형 조정
- 제조/물류 배경 validation 이미지 추가
- bbox 라벨 품질 점검
- `YOLOv8s` 등 더 큰 모델과 비교 실험
- augmentation 설정 조정
- 클래스별 precision, recall, mAP를 따로 확인하여 약한 클래스를 우선 보완

## 6. 발표용 요약

> YOLOv8n 파인튜닝 결과 Precision 0.87, Recall 0.72, mAP50 0.79를 달성하여 제조/물류 객체 탐지 프로토타입으로는 충분한 성능을 확인했다. mAP50-95는 0.56으로 상대적으로 낮았는데, 이는 여러 공개 데이터셋을 병합하면서 발생한 bbox 라벨링 기준 차이와 클래스 불균형의 영향으로 해석된다. 추후 forklift 데이터 추가 확보와 라벨 품질 개선을 통해 bbox 정밀도를 높일 계획이다.
